Describing semantics Elements in my Own words

In language semantics provide the meaning or value assigned to words and symbols. They differ from syntax which provides the structural framework for words as in sentences or paragraphs. Regarding HTML5, semantics elements share a similar definition. They provide a definition of what something is. A developer will use semantics elements to define certain parts of a page like the header, paragraphs, or body. They can also define content types like lists, summaries, or captions. The raw text is then human-readable and therefore shareable or editable. When the page is rendered by a browser the sematic elements are also important because they define the content and enable it to be displayed logically to an end user. semantics elements are related to syntactical elements more closely in the second example because without following the appropriate syntax, the browser will not be able to render them appropriately which would cause them to lose meaning.
